# data_sample — 영수증 샘플 데이터

OCR·분류·매칭 개발과 데모에 쓸 **영수증 샘플 이미지와 그 설명**을 모아두는 폴더입니다.
팀원 누구나 참고할 수 있도록 저장소에 함께 올립니다.

## 구조

```
data_sample/
├── README.md          이 문서 (샘플 목록표)
├── images/            영수증 이미지 원본
└── expected/          이미지별 OCR 기대 결과 JSON (파일명은 이미지와 동일)
```

## 등록 방법

1. 이미지를 `images/`에 넣습니다. 파일명은 `r001-스시로-신주쿠.jpg` 처럼 `번호-가맹점-특징` 형식으로 적습니다.
2. 아래 **샘플 목록** 표에 한 줄을 추가합니다.
3. OCR 기대 결과를 아는 경우 `expected/r001-스시로-신주쿠.json`에 적습니다. 형식은 [docs/02-API-CONTRACT.md](../docs/02-API-CONTRACT.md)의 `Receipt.ocr`을 따릅니다.

## ⚠️ 올리기 전 반드시 확인

[CONTRIBUTING.md](../CONTRIBUTING.md)의 "실제 영수증·법인카드 정보는 커밋하지 않습니다" 규칙이 적용됩니다.

- **카드번호는 뒤 4자리만 남기고 가립니다.** 이미지 위에 검은 박스로 덮은 뒤 저장하세요.
- 실명, 전화번호, 주소, 사업자 개인정보가 찍혀 있으면 가립니다.
- 개인 카드 영수증은 올리지 않습니다. 회사 법인카드 영수증도 실제 건이면 금액·가맹점 외의 식별정보는 가립니다.
- 이미지 한 장은 **2MB 이하**로 리사이즈합니다. (CONTRIBUTING: 대용량 바이너리 커밋 금지)
- 확실하지 않으면 올리기 전에 팀에 먼저 물어봅니다.

## 샘플을 고를 때

데모와 개발에 도움이 되도록 아래가 골고루 섞이면 좋습니다.

- 계정과목별로 최소 1장씩 (식대·교통·숙박·간식·도서·AI구독)
- 외화 영수증 (엔화 등) — 환율 환산 화면 검증용
- 인식이 어려운 것 1~2장 (구겨짐, 어두움, 흐림) — 예외 처리 시연용
- 카드 승인내역과 짝이 맞는 것 / 안 맞는 것 — 매칭 로직 검증용
