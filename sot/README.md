## 현재 상태 스냅샷

- 단계: 해커톤 MVP + 정산단위 엔진 백엔드(`presets.js`) 구현됨. 화면은 디자인 위주 진행 중 (`eaccounting/`, `design/`, `app/`).
- 핵심 결정 1: 출장모드(Trip)·복지비(Budget)를 별도 개념 대신 **정산단위**라는 단일 엔티티로 통합 (`02_architecture_db_schema.md`).
- 핵심 결정 2: **유입(촬영/업로드)만 모바일·앱, 정산·상신은 이어카운팅.** 정산단위 생성도 전부 이어카운팅에서 (해외출장=품의 자동생성 seed, 국내출장=직원 직접, 특수정산=관리자 배포). 모바일에는 생성 기능 없음.
- 핵심 결정 3: 정산은 반자동 — 정산단위 지정칸(계정과목·Cost Center·적요양식·전결라인 양식)은 자동완성·고정, 나머지는 일반 정산처럼 입력.
- 데모: 유형별 2케이스 (해외출장=모바일 촬영 / 비출장 AI구독=PC 업로드) — `demo/demo-cases.html`, `06_spec.md` 참고.
- 이번 라운드에서 안 하는 것: 실제 메일 발송, 품목별 비목 분리, 다중 사용자, 상신 하드 블록. (`01_project_purpose.md`의 Non-Goals 참고)
