# 정산 시스템 개선 요청사항 (개발자 전달용)

> 작성자: 전은아 (soulotion) · 브랜치: `feat/soulotion-settlement-improvement`
> 목적: 법인카드 정산 화면에 필요한 개선사항을 개발자에게 전달하기 위한 요구사항 정리

## 1. 회계팀 검토 전표 탭 - 결재라인 추가

**위치**: 법인카드 정산 화면 > 카드내역 개별 항목 선택 후 **정산하기** 클릭 시 나타나는 4개 탭 중 **회계팀 검토 전표** 탭 (화면 상단 타이틀: "법인카드 - (회계팀검토)")

**요구사항**:
- 회계팀 검토 전표 탭을 선택하면, 기존 결재선지정 표(기안/검토/승인) **아래에 한 줄이 통째로 추가**되어 회계팀 검토 라인이 들어가야 함. (기존 표의 검토/승인 칸에 끼워넣는 것이 아니라, 표 아래 새로운 행으로 추가)
- 추가 결재라인 (기본 결재라인 뒤에 순서대로 붙음):
  1. 회계팀 김행복 매니저 — **검토**
  2. 회계팀 조아요 팀장 — **승인**
- 나머지 3개 탭은 기존과 동일하게 기본 결재라인만 사용 (변경 없음).

**참고**: 기존 전결규정 데이터는 [fixtures/approval-rules.json](../fixtures/approval-rules.json)에 `category`별 `approvers` 배열로 정의되어 있음. 회계팀 검토 라인은 탭 선택(문서 유형)에 따라 붙는 것이므로, 카테고리 기반 규칙과는 별도로 "회계팀 검토 전표 탭 선택 시 추가 결재자"를 처리하는 로직이 필요.

## 2. 예산부서 콤보박스

**위치**: 카드 정산 화면의 예산부서 입력란

**요구사항**:
- 현재 텍스트 입력 방식인 예산부서 란을 **콤보박스(드롭다운 선택)** 로 변경.
- 선택 항목 (표시명 / 코드):
  | 부서명 | 코드 |
  |---|---|
  | 기업문화AX팀 | A341 |
  | 총무공통비용 | A861 |
  | 교육 | A860 |

**참고**: 현재 예산 관련 마스터 데이터는 [fixtures/budgets.json](../fixtures/budgets.json)에 있으나 부서 코드 체계가 없으므로, 위 3개 부서(코드 포함)를 신규 마스터 데이터로 추가해야 함.

## 3. 해외출장비 정산 - 결재선지정 직급 순서 오류

**위치**: 전불/출장비 > 해외출장비 정산 화면 > **결재선지정** 탭

**현재 상태 (오류)**:
- 기안: 매니저 (홍길동 팀원)
- 검토: 실장 (박아무개 실장)
- 승인: 팀장 (김아무개 팀장)

**요구사항**:
- 결재라인이 직급 순으로 정리되어야 함. 아래 순서로 수정 필요:
  1. 기안 — 매니저 (변경 없음)
  2. 검토 — **팀장**으로 변경
  3. 승인 — **실장**으로 변경
- 즉, 현재 "검토"와 "승인"에 배정된 결재자(실장/팀장)가 서로 뒤바뀐 상태이므로 순서를 맞바꿔야 함.

## 4. 법인카드 (현업완결) - 미리보기(기안) 팝업화

**위치**: 법인카드 - (현업완결) 화면 우측 상단 **미리보기(기안)** 버튼

**요구사항**:
- 현재 미리보기(기안) 버튼의 동작을 팝업(모달) 형태로 변경.
- 팝업에는 해당 전표의 전체 내용이 표시되어야 함: **결재선지정**부터 **전표정보, 예산정보, 카드이용내역, 첨부파일**까지 전 구간 포함.
- 사용자가 팝업에서 내용을 검토한 뒤, 팝업 내 **"기안"** 버튼을 눌러야 최종 상신되는 흐름으로 변경.
- 즉, 현재는 버튼명이 "미리보기(기안)"로 미리보기와 상신이 한 번에 묶여 있는 것으로 보이나, 미리보기(팝업 노출)와 기안(상신 확정)을 분리하여 사람이 검토 후 명시적으로 상신하는 단계를 추가해야 함.

## 5. 법인카드 (현업완결) - 결재선지정 "검토" 조직명 오류

**위치**: 법인카드 - (현업완결) 화면 > **결재선지정** 탭 > 검토 라인

**현재 상태 (오류)**:
- 검토: 경영기획실 최아무개 매니저

**요구사항**:
- 검토 라인의 조직명을 "경영기획실" → **"지원팀"** 으로 변경.
- 결재자(최아무개 매니저)는 그대로 유지, 소속 조직명만 수정.

## 6. 해외/국내출장비 정산 - 예산부서 기본값 및 콤보박스화

**위치**: 전불/출장비 > 해외출장비 정산, 국내출장비 정산 화면 > **예산정보** 탭의 예산부서 입력란

**요구사항**:
- 출장 시 결제한 카드/현금 내역을 불러올 때, 예산부서란에 **기안자(작성자)의 소속 부서**가 기본값으로 자동 반영되어야 함.
  - 예: 전력사업기획팀 소속 직원이 정산할 경우 예산부서에 "[AQ131]전력사업기획팀"이 디폴트로 표시.
- 단, 필요 시 다른 예산부서로 변경할 수 있어야 하므로 입력란을 텍스트 입력이 아닌 **콤보박스(드롭다운 선택)** 형태로 변경.
- 즉 동작 순서: ① 기안자 소속 부서 기준 기본값 자동 세팅 → ② 사용자가 콤보박스에서 다른 부서로 변경 가능.
- 참고: [2번 항목](#2-예산부서-콤보박스)의 법인카드 정산 화면 예산부서 콤보박스화와 동일한 개념이며, 이번 건은 해외/국내출장비 정산 화면에 적용 + 기안자 소속 기반 기본값 로직이 추가로 필요함.

## 7. 간편정산 상신 전표가 나의 문서함(결재문서)에 반영되지 않음

**위치**: 간편정산 > 자동매칭 > **정산·전표 생성** 화면 → **검토 완료·상신하기** 버튼 → 나의 문서함 > 결재문서 > **전체조회**

**현재 상태 (오류)**:
1. 간편정산에서 영수증 업로드 후, 자동매칭 메뉴에서 정산하고 싶은 항목을 선택 → 정산·전표 생성 화면에서 전표 초안 확인 → **검토 완료·상신하기** 클릭.
2. 상신 완료 화면에 "전표(예: vch_003, ₩200,435)가 접수되었습니다"라는 메시지와 함께 **"나의 문서함에서 확인 >"** 버튼이 표시됨.
3. 그러나 해당 버튼을 눌러 나의 문서함 > 결재문서 > 전체조회로 이동해도, 방금 상신한 전표가 목록에 **존재하지 않음**. UI는 확인 가능하다고 안내하지만 실제로는 조회되지 않는 상태.

**요구사항**:
- 간편정산(자동매칭 → 정산·전표 생성) 흐름을 통해 상신된 전표가 상신 즉시 나의 문서함 > 결재문서 > 전체조회(및 결재할문서 등 관련 목록)에 **정상적으로 노출**되어야 함.
- "나의 문서함에서 확인 >" 버튼과 실제 목록 데이터가 반드시 일치해야 하며, 상신 성공 메시지를 띄우면서 정작 조회가 안 되는 상태(현재 버그)가 재발하지 않도록 처리.

**참고**: 이 저장소의 목업 기준으로 보면, 전표 상신 API([server/routes/vouchers.js](../server/routes/vouchers.js) `POST /api/vouchers`)는 상신된 전표를 인메모리 store(`db.vouchers`)에 저장하지만, 나의 문서함 전체조회 화면([eaccounting/js/mydocs-data.js](../eaccounting/js/mydocs-data.js))은 `db.vouchers`를 조회하는 것이 아니라 **고정된 목데이터 배열(APPROVAL_DOCS, 29건)** 을 그대로 렌더링하는 구조. 실제 시스템에서도 유사하게 "상신 처리"와 "나의 문서함 목록 조회"가 서로 다른 데이터 소스/캐시를 참조하고 있어서 방금 상신한 전표가 누락되는 것으로 추정되므로, 상신 API가 반영하는 데이터와 나의 문서함 목록 조회 API가 같은 소스를 바라보도록 동기화하는 수정이 필요함.

## 8. 간편정산 메뉴명 변경

**위치**: 상단 GNB **간편정산** 메뉴

**요구사항**:
- 메뉴명 "간편정산" → **"증빙 보관함"** 으로 변경.
- 하위 메뉴(영수증 업로드, 자동매칭, 정산단위 설정, 분석 대시보드) 구성은 변경 없음, 상위 메뉴명만 수정.

## 9. 자동매칭 - 선택 항목 삭제 기능 추가

**위치**: 간편정산(→ 증빙 보관함) > **자동매칭** 화면 > 목록에서 항목 체크 시 나타나는 액션 바

**현재 상태**:
- 항목을 1건 이상 체크하면 액션 바에 **"선택 건으로 정산·전표 생성 >"** 버튼만 노출됨.

**요구사항**:
- 항목을 체크했을 때 **"선택 건으로 정산·전표 생성 >"** 버튼은 **숨김 처리**.
- 같은 위치에 **"선택 삭제"** 버튼을 새로 노출.
- "선택 삭제" 클릭 시, 현재 체크된 항목들이 목록에서 **삭제**되도록 처리.

## 10. 출장비 정산 - 카드이용내역에 영수증 첨부 누락

**위치**: 전불/출장비 > 해외출장비 정산(국내출장비 정산 동일) > **카드이용내역** 항목 / 나의 문서함 > 결재문서 > 전체조회에서 상신된 출장 전표 상세 화면

**현재 상태 (오류)**:
- 출장비 정산 시 출장기간에 해당하는 카드 이용내역은 정상적으로 불러와지지만, 각 이용내역 건에 **영수증(첨부파일)이 붙어있지 않음**.
- 동일한 문제가 나의 문서함 > 결재문서 > 전체조회에서도 재현됨 — 이미 상신된 출장 전표를 클릭해 세부내용을 봐도, 카드 이용내역 각 건에 대응하는 영수증이 하단에 표시되지 않음.

**요구사항**:
- 출장비 정산 화면에서 카드이용내역을 불러올 때, 각 이용내역 건에 매칭되는 영수증이 **함께 첨부되어 표시**되어야 함.
- 상신 후 나의 문서함에서 해당 전표를 다시 열람할 때도, 상신 시점의 영수증 첨부 상태가 그대로 **유지되어 조회**되어야 함 (정산 화면과 상신 후 조회 화면 모두 동일하게 반영).

## 11. 나의 문서함 - 결재문서 화면의 결재/반려 버튼 제거

**위치**: 나의 문서함 > 결재문서 > 전체조회 화면 상단 액션 영역의 **"✔ 결재"** / **"↺ 반려"** 버튼

**요구사항**:
- 위 두 버튼을 화면에서 **제거**.

**참고**: [eaccounting/mydocs-all.html](../eaccounting/mydocs-all.html)의 `approveChecked('결재완료')` / `approveChecked('반려')` 호출 버튼([mydocs-all.html:56-57](../eaccounting/mydocs-all.html))이 해당 버튼임.
