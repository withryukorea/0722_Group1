2. 정산 자동화 & 전표 접수

영수증 저장 이후의 정산(매칭 → 전표 생성 → 상신)은 자동으로 처리된다. Preset이 회계 항목을 이미 싣고 있으므로 정산 단계에서 사람이 손댈 것이 없다. 별도의 정산 입력 화면은 두지 않는다.

대시보드가 보여주는 것

  • 출장(TRIP): 일별 구분 없이 비목별 누적 현황. 특히 일당(per-diem) 한도 대비 사용/잔여를 중심으로 표시
  • 월 한도(RECURRING): 교육·도서 등 배정 예산의 사용/잔여 게이지
  • 영수증을 저장·정산하면 해당 Preset 사용량이 자동 반영되어 갱신

데이터 출처

이미 설계된 GET /api/presets?active=true 의 usage · rules.limitKRW 를 그대로 읽어 렌더링. 신규 API 없이 표시 화면만 추가.

정산 흐름과의 연결

이어카운팅에서 Preset을 배포 → 모바일에서 영수증 처리 시 자동 적용 → 사용량이 이 대시보드에 실시간으로 쌓인다. "규정을 배포하면 정산이 알아서 따라간다"는 구조를 사용자가 눈으로 확인하는 화면.
