# 정산 기능 개선 논의 정리

> 작성자: 전은아 (soulotion) · 브랜치: `feat/soulotion-settlement-improvement`
> 목적: 법인카드 영수증 자동 전표화 시스템에서 추가로 필요한 기능/고려사항을 논의하고 팀 공유용으로 정리

## 1. 업종 미스매치 시 증빙 첨부

식대성 경비로 처리하려면 가맹점 업종이 음식점/카페 등으로 잡혀야 하는데, 간혹 카드 시스템에 **페이, 전자상거래, 키오스크** 등 다른 업종으로 표시되는 경우가 있음.

- 업종 코드만으로 식대 여부를 판단하지 말고, 업종 미스매치 시 반려 대신 **증빙 첨부 요청 상태**를 별도로 두는 예외 처리 플로우 필요.

## 2. 계정/예산부서 일괄 입력

동일한 계정과목·예산부서를 여러 건에 반복 입력하는 것이 번거로움.

- 여러 행을 다중 선택한 뒤 계정/예산부서를 **일괄 적용**하는 기능 필요.

## 3. 부가세 자동 인식

영수증에 있는 부가세를 정확히 자동 인식해야 함.

- 세금계산서/신용카드매출전표는 공급가액·부가세가 별도 표기되어 OCR로 비교적 정확히 추출 가능.
- 간이영수증, 부가세 면세 업종(일부 식음료 포함)은 부가세 항목이 없거나 합계만 표기되어 역산(합계/11)에 의존해야 하며 오차 가능성이 있음 → 검토 단계에서 오차 가능성 노출 필요.
- **신용카드 영수증이라도 부가세를 정확히 인식하지 못하는 경우가 있으므로 유의.** OCR 신뢰도(confidence)가 낮으면 자동확정 대신 "부가세 확인 필요" 플래그를 검토자에게 노출하는 방식 필요.

## 4. 현금성 결제(상품권) 결재라인

배달의민족 5만원권, 스타벅스 3만원권 등 현금성 상품권 결제 건은 **차상위자 결재**가 필요.

- 결제수단 유형만으로 자동 판별하여 차상위자 결재라인을 강제 추가하는 규칙 필요.

## 5. AI 구독료 정산 템플릿

AI 구독료는 결재라인에 현업 검토가 들어가도록 정해져 있고, 정산 시 예산부서/계정/고정 검토자/적요(예: `[AI구독료] 구독 AI 이름_구성원 이름`)가 자동 세팅되면 좋음.

- **주의**: AI 구독료를 개인 자기개발비 예산으로 처리하는 경우도 있어, 모든 AI 구독료 건을 자동으로 템플릿 처리하면 안 됨.
- 결론: 자동판단(가맹점/적요 패턴 매칭)으로 강제 세팅하지 않고, **"AI구독료 정책 적용" 버튼/토글**을 두어 사람이 명시적으로 클릭했을 때만 예산부서·계정·고정검토자·적요가 자동 채워지는 방식으로 결정.
- 보완 아이디어: 시스템이 패턴은 감지하되 버튼을 강조 표시(추천)만 하고, 최종 적용 여부는 사람이 클릭으로 결정하는 절충안.

## 6. 전결규정 기반 알람

전결규정 파일을 참고하여, 실제 결재라인이 규정에서 벗어나면 알람이 뜨도록 함.

- 전결규정 파일을 규칙엔진화하여 실제 결재라인과 비교 후 이탈 시 알람.
- 전결규정 파일의 실제 형태(표/텍스트 등)를 확인 후 파싱 방식 결정 필요 (TODO).

## 7. 출장비 일당 실시간 소진 현황

출장비 정산 시 일당이 정해져 있는데, 정산 진행에 따라 일당에서 얼마나 차감됐는지 실시간으로 보이면 좋음.

- 출장비는 일당 총액에서 정산 건별로 실시간 차감되는 **잔액 트래커**(사람별/출장건별)가 필요.

## 8. 영수증 자동 매칭 및 계정/적요 자동 반영 (다건 처리)

출장 후 영수증이 많아 분류·계정 입력이 번거로움. 영수증 사진을 업로드하면:

1. 결제일시, 승인번호, 카드정보, 사용내역, 업체이름 등을 인식
2. 정산 시스템의 각 카드 결제내역에 증빙이 **자동으로 첨부**
3. 인식 내용을 바탕으로 정해진 분류에 맞게 **계정과목·적요 자동 반영**
4. 사람은 검토 후 바로 기안

예시: 호텔 결제 영수증에 숙박비/세탁비/조식비가 함께 있으면 숙박비는 숙박비 계정, 세탁비/조식비는 일당 계정으로 자동 분리.

논의된 두 단계:

- **매칭**: 승인번호를 최우선 매칭 키로 사용(카드사 데이터에도 승인번호 존재 → 1:1 매칭 가능). 승인번호 인식 실패 시 날짜+금액+카드 뒷자리 조합으로 폴백.
- **분류(항목 분리 포함)**: 업체명/업종 기준 통짜 분류(예: 카페→식대)는 상대적으로 쉬움. 다만 한 영수증 내에서 항목별로 계정을 나눠야 하는 경우(호텔 예시)는 라인아이템 단위 OCR과 항목명→계정 매핑 키워드 사전이 필요해 난이도가 높음(호텔마다 "Laundry Service"/"세탁"/"Valet" 등 표기가 다름).
  - **결정**: 항목분리는 적극적으로 자동화하지 않는다. 신뢰도와 무관하게 통짜로 하나의 계정에 넣은 뒤 "항목 분리 필요" 플래그만 띄워 검토자가 수동으로 나누게 하는 방식으로 확정.
